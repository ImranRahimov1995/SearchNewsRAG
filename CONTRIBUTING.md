# Contributing Guidelines

Thank you for your interest in contributing to this project!
This project is open to contributions but follows a strict
non-commercial usage policy. Please read the guidelines below
before submitting any changes.

## 1. General Principles

- By contributing, you agree to the project's license terms.
- All contributions (code, documentation, ideas) submitted through
  Pull Requests become part of the project and are distributed under
  the **Open Source Non-Commercial (OSNC)** license.
- Contributions intended for commercial usage, monetization, or
  integration into revenue-generating products will not be accepted.

## 2. How to Contribute

1. Fork the repository.
2. Create a new branch:

git checkout -b feature/your-feature

3. Make your changes.
4. Ensure everything builds, runs, and passes tests.
5. Open a Pull Request describing your changes clearly.

## 3. Code Requirements

- Write clean and maintainable code.
- Follow the existing project style (e.g., PEP8, ESLint, etc.).
- Use SOLID principles where applicable.
- Add comments when the logic is non-obvious.
- Do not include temporary files, IDE artifacts, or unnecessary dependencies.

## 4. Tests

- Add or update tests if the project uses them.
- Do not break existing functionality.
- If no tests exist, verify your changes manually.

## 5. Discussions & Proposals

Before working on large or architectural changes,
open an Issue to discuss your idea.
This helps avoid wasted effort and ensures alignment with project goals.

## 6. Forbidden Contributions

- Changes that enable or promote commercial use.
- Attempts to bypass or weaken the OSNC license.
- Adding external code that violates another license or copyright.

## 7. Copyright

By submitting a contribution, you confirm that:

- You are the original author of the code.
- Your contribution does not violate third-party licenses.
- You grant the project the right to use and distribute your contribution
under the OSNC license.


Thank you for helping improve the project!
